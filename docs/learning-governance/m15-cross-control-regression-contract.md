# M15 Track D cross-control regression contract

Track D is a deterministic, offline, read-only, additive, contract-first, simulation-only regression package. It composes caller-supplied inert mappings to prove that evidence introduced by M15 Tracks A-C cannot bypass independently governed AAOS controls. It imports or executes no referenced artifact and changes no Track A, B, or C contract.

## Governing separations

Learning Proof is not retention approval, training approval, memory persistence, skill admission, runtime execution, or Learning Proof sealing. Capability Memory Pack is not package admission, tool registration, executable capability, or runtime authorization. Rollback readiness is not rollback authorization or execution. Deletion evidence is not deletion authorization, physical erasure, or provider-side deletion. Portability evidence is not provider migration approval, replacement-model authorization, or production execution.

AI provenance is not identity proof, human review, or merge approval. Public evidence is not public-output authorization or permission to expose sensitive material. Decision Proof linkage is not verification or sealing. Learning Proof sealing and Decision Proof sealing remain AAOS-owned. AAOS remains the decision sovereignty layer.

## Control domains

The machine-readable matrix binds Tracks A-C and cross-track composition to public-output safety; provenance and human review; skill, MCP package, tool, and capability admission; evidence extraction, freshness, supersession, and revocation; governance and execution authority; Decision Proof ownership; and M15/release publication state.

Sensitive-output inspection applies recursively to every mapping key and scalar value, including extensions, evidence references, identifiers, reason codes, review and provenance metadata, graph nodes, and capability manifests. Schema validity, trusted workflow origin, Decision Proof linkage, synthetic classification, verification, and portability never exempt inspection. Findings identify locations and categories but never echo detected values.

## Evaluation contract

The evaluator accepts caller-supplied inert mappings only. It performs no network access, file access, subprocess or shell execution, dynamic external import, package installation, MCP registration, tool execution, model invocation, file mutation, rollback, deletion, provider operation, or external state transition. It does not mutate input. Malformed, cyclic, deeply nested, shared-container, and hostile inputs fail safely. Findings are deterministic, sorted, and deduplicated.

`pass` means only that the supplied synthetic evidence preserved the tested boundaries. `quarantine` is used for sensitive-output findings. `reject` is used for all other boundary violations. No result grants governance, policy, identity, risk, deployment, installation, package admission, tool registration, runtime, rollback, deletion, migration, replacement-model, audit, waiver, sealing, milestone-completion, or release authority.

## Release boundary

Track D does not close tracker #231, declare M15 complete, approve completion review, finalize release readiness, authorize v0.14.0, create a tag, or create a GitHub Release. Tracker #231 remains Open. M15 remains active and incomplete. v0.14.0 remains unpublished.

All fixtures are synthetic and contain no real credentials, tokens, personal information, medical or financial identifiers, confidential prompts, private evaluations, provider-managed memory, or production account information.
