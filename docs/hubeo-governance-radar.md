# AAOS Governance Radar

AAOS Governance Radar is the public external signal admission layer for AAOS.

It is not a tool directory, model registry, scanner catalog, marketplace index, or agent framework list.

External tools, scanners, runtimes, models, MCP servers, GitHub Actions, and agent frameworks may enter AAOS only as governed signals.

They may become:

- evidence candidates
- admission records
- replay inputs
- audit inputs
- future adapter issues
- radar nodes

They must not become:

- governance authorities
- policy authorities
- identity authorities
- decision routers
- release approvals
- audit closures
- risk calibration owners
- fail-closed rule owners

## HUBEO

HUBEO means:

- Hub: external signal intake point
- Evidence: evidence candidate normalization
- Observatory: governance radar and trend detection
- Operations: adapter routing and issue workflow

## Layer Rule

External sources may map into AAOS layers, but they must not collapse AAOS layers.

Allowed candidate mappings:

- L3: model / runtime candidate
- L4: tool / data execution candidate
- L4.5: evidence adapter candidate
- L5: memory / reconstruction candidate
- L6: verification / audit input candidate

L0, L1, and L2 remain AAOS sovereign governance layers.
