# Adapter Intake Taxonomy

AAOS adapter intake classifies external sources before they become governance adapter issues.

## Source Types

- model-runtime
- local-runtime
- distributed-inference
- scanner
- github-action
- mcp-server
- agent-framework
- repository-intelligence
- supply-chain-tool
- audit-tool
- web-audit
- memory-source
- reconstruction-candidate
- other

## Candidate Layer Routing

- L3: model runtime or inference runtime
- L4: tool, data, API, device, or runtime execution substrate
- L4.5: evidence adapter
- L5: memory, reconstruction, topology, dependency, or repository-history candidate
- L6: verification, replay, audit input, or audit subject

## Required Boundary Statement

Every intake record must include a not-authority statement.

Example:

```text
This source is an evidence candidate, not a governance authority.
```
